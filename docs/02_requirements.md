# Этап 2. Требования (функциональные)

## Обязательные требования (MUST)

Эти функции реализованы в MVP и являются критическими для работы системы.

**REQ-1. Аутентификация сотрудника**
Пользователь должен иметь возможность войти в систему, используя корпоративную почту и пароль.
*   **Критерий приёмки:**
    1. При вводе корректных данных сервер возвращает профиль пользователя и токен доступа.
    2. При вводе некорректных данных появляется сообщение "Ошибка авторизации".
    3. Доступ к внутренним страницам без авторизации невозможен (редерект на Login).

**REQ-2. Просмотр Канбан-доски**
Пользователь видит все задачи выбранного проекта, сгруппированные по статусам: `TODO`, `PROGRESS`, `DONE`.
*   **Критерий приёмки:**
    1. Задачи отображаются в соответствующих колонках.
    2. Карточка задачи содержит: заголовок, приоритет (цветом), аватар исполнителя, дедлайн.
    3. При переключении проекта в шапке (фильтр) список задач обновляется без перезагрузки страницы.

**REQ-3. Создание новой задачи**
Пользователь может создать новую задачу через модальное окно.
*   **Критерий приёмки:**
    1. Нажатие кнопки "+" открывает модальное окно.
    2. Поля "Название", "Приоритет", "Дедлайн" обязательны.
    3. После нажатия "Создать" задача немедленно появляется в колонке `TODO`.

**REQ-4. Смена статуса задачи**
Пользователь может переводить задачу на следующий этап жизненного цикла.
*   **Критерий приёмки:**
    1. Клик по кнопке действия (или перетаскивание) меняет статус: `TODO` -> `PROGRESS` -> `DONE` -> `TODO`.
    2. Изменение сохраняется в базе данных.
    3. Цвет индикатора статуса обновляется.

**REQ-5. Реестр сотрудников (Поиск)**
Пользователь может просматривать список коллег и искать их.
*   **Критерий приёмки:**
    1. Список загружается постранично или полностью.
    2. Ввод текста в поле поиска мгновенно фильтрует таблицу по совпадению в полях: Имя, Email, ID.

**REQ-6. Аналитический Дэшборд**
На главном экране отображается сводная статистика.
*   **Критерий приёмки:**
    1. Отображается общее количество сотрудников и задач.
    2. Отображается список "Последние задачи" (Top-5 свежих).
    3. Данные соответствуют текущему состоянию БД.

---

## Желательные требования (NICE)

**REQ-7. Разграничение прав (Roles)**
Только пользователи с ролью "Admin" или "Supervisor" могут удалять задачи. Обычные сотрудники могут только менять статус.

**REQ-8. Drag-and-Drop**
Смена статуса задачи осуществляется перетаскиванием карточки между колонками, а не только кликом.

**REQ-9. Комментарии к задаче**
Возможность оставить текстовый комментарий внутри карточки задачи для обсуждения деталей.
