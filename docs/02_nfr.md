# Нефункциональные требования (NFR)

## 1. Производительность и отзывчивость (Performance)
*   **Отклик API:** Время ответа сервера на запросы чтения (GET `/api/tasks`) не должно превышать **500 мс** при нормальной нагрузке.
*   **Отклик UI:** Рендеринг списка задач (до 100 элементов) на клиенте должен занимать не более **200 мс**.
*   **SPA Navigation:** Переход между разделами (Dashboard -> Kanban) должен происходить мгновенно (< 100 мс) без перезагрузки страницы.

## 2. Надёжность и доступность (Reliability)
*   **Обработка ошибок:** При падении API или отсутствии сети клиентское приложение не должно "падать" (белый экран). Должно отображаться уведомление (Toast) или заглушка ошибки.
*   **Валидация данных:** Все формы (Login, Create Task) должны иметь валидацию на клиенте и на сервере.

## 3. Хранение данных (Storage)
*   **СУБД:** PostgreSQL 16+.
*   **ORM:** Entity Framework Core (Code First подход).
*   **State:** Состояние интерфейса (выбранный проект, открытая вкладка) не сохраняется при закрытии вкладки браузера, но сохраняется токен авторизации (JWT).

## 4. Ограничения платформы (Constraints)
*   **Бэкенд:** ASP.NET Core 10.0 (C#).
*   **Фронтенд:** React 18 + Vite.
*   **Браузеры:** Поддержка последних версий Chrome, Edge, Firefox, Safari.
*   **Экран:** Оптимизация под Desktop (минимальная ширина 1280px, согласно дизайну Island UI).

## 5. Безопасность (Security)
*   **Аутентификация:** Использование JWT (Access Token). Токен передается в заголовке `Authorization: Bearer ...`.
*   **Пароли:** Хранение паролей в БД только в хэшированном виде (используя Identity или BCrypt).
*   **CORS:** API должно принимать запросы только с разрешенных доменов клиента (localhost в разработке).
