# 1. Название проекта
**Splitflow** — Корпоративная система управления задачами и проектами.

# 2. Описание
Splitflow представляет собой веб-приложение для автоматизации управления проектами внутри организации. Система позволяет создавать, распределять и отслеживать задачи в режиме реального времени, используя визуализацию процесса (Канбан-доска) и аналитические инструменты. Решение направлено на повышение прозрачности рабочих процессов и улучшение координации между отделами.

# 3. Пользователи
- **Руководители проектов (Супервайзеры):** Используют систему для создания задач, назначения исполнителей, мониторинга загруженности команды через аналитический дэшборд и контроля сроков выполнения.
- **Сотрудники (Исполнители):** Используют систему для получения списка своих задач, изменения их статусов (Todo -> Progress -> Done) и просмотра информации о коллегах.

# 4. Ключевой функционал (MVP + улучшения)
В первую версию системы входят:
- **Аутентификация и авторизация:** Безопасный вход по корпоративной почте, разделение прав доступа.
- **Интерактивная Канбан-доска:** Визуальное управление жизненным циклом задачи (создание, перемещение по статусам, удаление).
- **Управление задачами (CRUD):** Создание карточек с приоритетами (High/Medium/Low), дедлайнами и привязкой к конкретным проектам («Наследие», «Защита» и др.).
- **Реестр сотрудников:** Список персонала с функцией мгновенного поиска (по ID, имени, почте) и просмотром KPI/уровня допуска.
- **Аналитический Дэшборд:** Сводная панель с метриками активности и загруженности по проектам.
- **Глобальная фильтрация:** Возможность переключения контекста приложения между разными проектами.

# 5. Что не входит в первую версию (Out of Scope)
- **Мобильное приложение:** Разрабатывается только веб-версия (адаптивная, но не нативная).
- **Хранение файлов:** Загрузка документов, изображений и макетов непосредственно в задачи не предусмотрена (только ссылки).
- **Календарь и Гант:** Отсутствует представление задач в виде календарной сетки или диаграммы Ганта.
- **Уведомления (Push/Email):** Система не рассылает письма или пуш-уведомления при изменении статуса задачи.
